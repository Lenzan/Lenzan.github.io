Git 学习笔记下

发表于 2018-06-07 | 分类于 Git |

git 标签

  • 轻标签
    • 添加名称
  • 注解标签
    • 添加名称
    • 添加注解
    • 添加签名

一般情况下，发布标签采用的注解标签来添加注解或者标签。
添加轻标签的命令 

git tag <tagName>

查看标签 

git tag

在 log 命令添加 –decorate 选项执行, 可以显示包含标签的提交记录。 

git log --decorate

添加注解标签 

git tag -a <tagName>
git tag -am "这里是注解" <tagName>

显示标签的列表和注解 

git tag -n

删除标签 

git tag -d <tagName>

git 改写提交

  • 修改最近的提交记录 

      git commit --amend

  • 取消过去的提交 

      git revert HEAD (这里是取消最近的一次提交)

  • 遗弃提交 reset
    在这行 reset 的时候根据影响的范围选择不同的模式。
    主要使用的场合：

    1. 复原修改过的索引的状态(mixed)
    2. 彻底取消最近的提交(hard)
    3. 只取消提交(soft)

    命令：删除最近一次提交 

      git reset --hard HEAD~

    在 reset 错误的时候可以执行下面的命令来回到 reset 之前的状态。 

      git reset --hard ORIG_HEAD

    详细表格：

模式名称 HEAD的位置 索引 工作树
soft 修改 不修改 不修改
mixed 修改 修改 不修改
hard 修改 修改 修改

  • 复制提交
    从其他的分支上将某次提交导入到 master 分支上，执行以下命令： 

    git checkout master   

git cherrypick <hash>

    如果有冲突修改就好。

  • 修改提交

      git rebase -i <commit>

    再次提交会显示提交的清单，在提交清单里找出要修改的提交，将”pick”改成”edit”,然后保存。等到文件修改后执行： 

      git commit --amend

    最后,指定 –continue 选项来执行 rebase。 

      git rebase --continue

  • 汇合分支上的提交,然后合并到分支
    这一操作要用到 merge 的特殊选项 squash
    操作命令：

      git checkout master  
  git merge --squash <otherbranch>

    这样 otherbranch 分支上的所有提交都合并到 master 分支上了。
